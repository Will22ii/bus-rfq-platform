# UI Specification
Shuttle Partner Beta RFQ System

본 문서는 RFQ 플랫폼의 UI 설계를 정의한다.

목표

- 업무용 웹 UI
- 표 기반 데이터 중심 인터페이스
- RFQ 생성 → 견적 제출 → 비교 → 선택 흐름 명확화

모바일은 고려하지 않으며 **데스크탑 웹 UI만 설계한다.**

# 2. 상태 표시 (UI)

UI에서 사용자에게 표시되는 상태

| 코드 | UI 표시 |
|---|---|
| open | 견적 접수중 |
| in_review | 심사 진행중 |
| completed | 선택 완료 |
| cancelled | 요청 취소 |

상태는 **색상 배지로 표시**

예


[견적 접수중] 파란색
[심사 진행중] 주황색
[선택 완료] 초록색
[요청 취소] 회색

# 11. UI 설계 요약

이 UI 구조는 다음을 보장한다.

- 명확한 RFQ 생성 흐름
- Supplier sealed bidding 유지
- Requester 비교 기능 강화
- 노선 기준 가격 정책 반영
- 단순한 Supplier 제출 모델
